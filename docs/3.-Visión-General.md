```
❌ Antes: public String nombre;

**Beneficio:** Control total sobre los datos

```
❌ Antes: Sin validación
✅ Después: Validación en setters + constructores

**Beneficio:** Datos siempre válidos

## 🎓 Lecciones Principales

### Lección 1: Refactorización es Iterativa
No se hace todo de una vez. Se hace paso a paso.

**Ejemplo:**
1. Separa programa principal
2. Luego separa responsabilidades
3. Luego agrega validación
4. Y así sucesivamente...

---

### Lección 2: Los Pequeños Detalles Importan
Constantes, nombres de métodos, formateo... todo contribuye.

**Ejemplo:**
```java
// ❌ Malo
public void imprDetalle() { ... }

// ✅ Bueno
public void imprimirDetalle() { ... }
```

---

### Lección 3: La Escalabilidad es Crítica
Pequeñas decisiones ahora = diferencias enormes después.

**Ejemplo:**
HashMap vs ArrayList con 1,000,000 items:
- ArrayList: ~1 segundo
- HashMap: ~0.001 segundos

---

### Lección 4: Documentación es Código
Si no está documentado, no existe.

**Ejemplo:**
- Javadoc para otros desarrolladores
- README para contexto
- Wiki para aprendizaje
- Comentarios para lógica compleja

---

### Lección 5: Validación es Defensa
Validar en entrada es mejor que intentar recuperarse después.

**Ejemplo:**
```java
// ✅ Validar en el setter
public void setPrecio(double precio) {
    if (precio < 0.01 || precio > 1_000_000.00)
        throw new IllegalArgumentException("Inválido");
    this.precio = precio;
}
```

## 🔄 Flujo de Datos

### Crear un Item
```
new Item("Camiseta", 20.99, 2)
    ↓
Validar nombre (no vacío)
    ↓
Validar precio (0.01-1M)
    ↓
Validar cantidad (>0)
    ↓
Item válido creado ✅
```

### Agregar al Carrito
```
carrito.agregarItem(item)
    ↓
¿Existe el item?
    ├─ Sí → Sumar cantidad
    └─ No → Agregar nuevo
    ↓
HashMap actualizado ✅
```

### Imprimir Detalle
```
carrito.imprimirDetalle()
    ↓
Para cada item en HashMap
    ├─ Imprimir item.toString()
    └─ Acumular total
    ↓
Mostrar total con moneda ✅
```
