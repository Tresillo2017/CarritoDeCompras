## Preguntas sobre Compilación y Ejecución

### ¿Cómo compilo el código?

```bash
javac src/*.java
```

Ver detalles completos en [[2.-Compilación-y-Ejecución|Compilación y Ejecución]].

---

### ¿Cómo ejecuto el programa?

```bash
java -cp src Main
```

La opción `-cp src` indica al Java dónde encontrar las clases compiladas.

---

### ¿Necesito instalar algo especial?

Solo Java Development Kit (JDK) versión 8 o superior.

**Descargar desde:** https://www.oracle.com/java/technologies/downloads/

---

### ¿Por qué dice "javac: command not found"?

Java no está instalado o no está en tu PATH.

**Solución:**
1. Instala JDK desde el enlace anterior
2. Reinicia tu terminal
3. Verifica con `java -version`

---

### ¿Cuánto tarda en compilar?

Menos de 1 segundo en una máquina moderna.

## Preguntas sobre Funcionalidad

### ¿Cuál es el rango válido de precios?

**Mínimo:** €0.01  
**Máximo:** €1,000,000.00

Fuera de este rango, lanza excepción:
```java
Item item = new Item("Producto", -50.00);
// IllegalArgumentException: El precio debe estar entre 0.01€ y 1000000.00€
```

---

### ¿Cuál es la cantidad mínima de un item?

**Mínimo:** 1 unidad

No puedes crear items con cantidad 0 o negativa:
```java
Item item = new Item("Producto", 50.00, 0);
// IllegalArgumentException: La cantidad debe ser mayor que 0
```

---

### ¿Puedo crear un item con nombre vacío?

**No.** Lanza excepción:
```java
Item item = new Item("", 50.00);
// IllegalArgumentException: El nombre del artículo no puede ser nulo o vacío
```

---

### ¿Qué pasa si agrego dos items con el mismo nombre?

Se suma automáticamente la cantidad:
```java
carrito.agregarItem(new Item("Camiseta", 20.99, 2));
carrito.agregarItem(new Item("Camiseta", 20.99, 3));
// Resultado: 1 item "Camiseta" con cantidad 5
```

Ver ejemplo detallado en [[7.-Ejemplos-de-Uso|Ejemplos de Uso]].

---

### ¿Puedo remover un item que no existe?

**No.** Lanza excepción clara:
```java
carrito.removerItem("Inexistente");
// IllegalArgumentException: El artículo 'Inexistente' no existe en el carrito
```

Esto es una característica, no un bug. Permite detectar errores.

---

### ¿Cómo actualizo la cantidad de un item?

```java
carrito.actualizarCantidad("Camiseta", 5);
```

Si el item no existe, lanza excepción.

---

### ¿Cómo obtengo el total del carrito?

```java
double total = carrito.calcularTotal();
System.out.println("Total: €" + String.format("%.2f", total));
```

---

### ¿Cómo obtengo la cantidad total de items?

```java
int cantidad = carrito.obtenerCantidadTotal();
// Retorna la suma de todas las cantidades
```

---

### ¿Cómo obtengo el número de artículos diferentes?

```java
int numArticulos = carrito.obtenerNumeroArticulos();
// Retorna el número de tipos diferentes de items
```

## Preguntas sobre Rendimiento

### ¿Por qué usas HashMap en lugar de ArrayList?

**HashMap es más eficiente:**

| Operación | ArrayList | HashMap |
|-----------|-----------|---------|
| Buscar | O(n) | O(1) |
| Remover | O(n) | O(1) |
| Actualizar | O(n) | O(1) |

Con 1,000,000 items:
- ArrayList: ~1 segundo por remover
- HashMap: ~0.001 segundos por remover

**1000x más rápido.**

Ver análisis completo en [[6.9.-Paso-9-Escalabilidad|Paso 9: Escalabilidad]].

---

### ¿Cuántos items puede manejar?

Teóricamente, millones. El rendimiento es consistente gracias a HashMap O(1).

---

### ¿Cuál es el uso de memoria?

Aproximadamente 50 bytes por item (varía según JVM).

Para 1,000,000 items: ~50 MB

## Preguntas sobre Arquitectura

### ¿Por qué hay 3 clases?

**Separación de responsabilidades:**
- **Item:** Modelo de datos
- **CarritoDeCompras:** Lógica del carrito
- **Main:** Programa principal

Ver detalles en [[2.-Separación-de-Responsabilidades|Mejora 2]].

---

### ¿Qué son principios SOLID?

5 principios de diseño:
1. **S**ingle Responsibility - Una responsabilidad por clase
2. **O**pen/Closed - Abierto para extensión, cerrado para modificación
3. **L**iskov Substitution - Comportamiento predecible
4. **I**nterface Segregation - Métodos específicos
5. **D**ependency Inversion - Usar abstracciones

Este proyecto demuestra los 5. Ver en [[12.-Principios-SOLID|Principios SOLID]].
