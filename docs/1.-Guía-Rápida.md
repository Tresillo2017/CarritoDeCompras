## ⚡ 5 Minutos para Entender CarritoDeCompras

### ¿Qué es?
Un carrito de compras en Java refactorizado con 10 mejoras profesionales.

### ¿Por qué importa?
Demuestra cómo transformar código básico en código profesional que:
- ✅ Es más eficiente (HashMap O(1) vs ArrayList O(n))
- ✅ Es más seguro (validación exhaustiva)
- ✅ Es más mantenible (separación de responsabilidades)
- ✅ Sigue principios SOLID

## 💻 Uso Básico (1 minuto)

### Crear un carrito
```java
CarritoDeCompras carrito = new CarritoDeCompras();
```

### Agregar items
```java
carrito.agregarItem(new Item("Camiseta", 20.99, 2));
carrito.agregarItem(new Item("Pantalones", 34.99, 1));
```

### Mostrar carrito
```java
carrito.imprimirDetalle();
// Salida: Lista formateada con total
```

### Actualizar cantidad
```java
carrito.actualizarCantidad("Camiseta", 5);
```

### Remover item
```java
carrito.removerItem("Pantalones");
```

### Obtener información
```java
carrito.calcularTotal();           // €166.97
carrito.obtenerCantidadTotal();    // 4
carrito.obtenerNumeroArticulos();  // 3
```

## 🔒 Validación Automática (1 minuto)

### ✅ Válido
```java
Item item = new Item("Laptop", 999.99, 1);  // ✅ OK
```

### ❌ Inválido - Precio
```java
try {
    Item item = new Item("Producto", -100.00);
} catch (IllegalArgumentException e) {
    System.out.println(e.getMessage());
    // "El precio debe estar entre 0.01€ y 1000000.00€"
}
```

### ❌ Inválido - Nombre
```java
Item item = new Item("", 50.00);  // ❌ Excepción
```

### ❌ Inválido - Cantidad
```java
Item item = new Item("Producto", 50.00, -5);  // ❌ Excepción
```

## 🎓 Conceptos Importantes (2 minutos)

### Encapsulación
Atributos privados con getters/setters públicos:
```java
// Privado - No se puede acceder directamente
private double precio;

// Público - Se accede vía getter
public double getPrecio() { 
    return precio; 
}

// Setter con validación
public void setPrecio(double p) {
    if (p < 0.01 || p > 1_000_000.00)
        throw new IllegalArgumentException("Precio inválido");
    this.precio = p;
}
```

### Responsabilidad Única
Cada método hace una cosa:
```java
// ✅ Bien: Responsabilidad única
public double calcularTotal()
public void imprimirListaArticulos()
public void imprimirTotal()

// ❌ Mal: Responsabilidades mezcladas
public void imprimirDetalleYCalcularTotal()
```

### Bajas Acoplamiento
Las clases no dependen una de la otra:
```java
// ❌ Alto acoplamiento (antes)
System.out.println(item.nombre + " - " + item.precio);

// ✅ Bajo acoplamiento (después)
System.out.println(item.toString());  // Item controla su formato
```
