# 0. Introducción

## 🎯 ¿Qué es CarritoDeCompras?

**CarritoDeCompras** es un proyecto educativo que demuestra cómo refactorizar y mejorar un código Java básico siguiendo principios profesionales de ingeniería de software.

## 📚 Contexto del Proyecto

### Punto de Partida
Se comenzó con un código simple de un carrito de compras que contenía:
- ❌ Responsabilidades mezcladas
- ❌ Programa principal acoplado a la lógica
- ❌ Atributos públicos sin validación
- ❌ Código duplicado
- ❌ Sin manejo de errores
- ❌ Estructura de datos ineficiente
- ❌ Sin documentación

### Transformación
A través de 10 mejoras sistemáticas, se convirtió en:
- ✅ Código profesional con separación clara de responsabilidades
- ✅ Arquitectura escalable y mantenible
- ✅ Validación exhaustiva de datos
- ✅ Manejo robusto de excepciones
- ✅ Documentación completa
- ✅ Ejemplos de uso
- ✅ Siguiendo principios SOLID

## 🎓 Objetivos Educativos

Este proyecto demuestra:

### 1. **Refactorización Sistemática**
Cómo mejorar código existente de manera ordenada y metódica.

### 2. **Principios SOLID**
Aplicación práctica de:
- **S**ingle Responsibility Principle
- **O**pen/Closed Principle
- **L**iskov Substitution Principle
- **I**nterface Segregation Principle
- **D**ependency Inversion Principle

### 3. **Mejores Prácticas de Java**
- Encapsulación completa
- Getters y setters con validación
- Métodos bien nombrados y enfocados
- Javadoc profesional

### 4. **Escalabilidad**
Cómo elegir estructuras de datos eficientes (HashMap vs ArrayList)

### 5. **Documentación Profesional**
Cómo documentar código de manera clara y exhaustiva

## ❓ Preguntas Iniciales

**¿Necesitas cambiar el código original?**  
Sí, pero de manera ordenada. Ver [[5.-Las-10-Mejoras|Las 10 Mejoras]].

**¿Es código para producción?**  
El carrito está completamente funcional, pero es principalmente educativo. Ver [[16.-Mejoras-Futuras|Mejoras Futuras]].

**¿Qué lenguaje se usa?**  
Java 8+. El código utiliza librerías estándar únicamente.

**¿Hay dependencias externas?**  
No, solo librerías estándar de Java (java.util).

**¿Puedo reutilizar este código?**  
Sí, es completamente independiente y fácil de adaptar.
